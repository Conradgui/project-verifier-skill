# project-verifier 定位拓展迭代

## Goal

将 `$project-verifier` 从“质量验证 + 面试证据”拓展为“项目理解、架构梳理、安全审计、质量验证、证据沉淀”的一体化 workflow skill。

## Scope

- 保持 skill 名称、目录名、安装路径和 `$project-verifier` 调用名不变。
- 更新 README、`SKILL.md`、Phase 2 workflow 和 Codex UI 默认提示。
- 新增固定项目理解文档包契约，不替代 README 副本和 verification workbench。

## Acceptance Checks

- README 明确出现 Project Understanding / 项目理解定位。
- README 明确区分 `project_understanding/`、`project_verification_workbench/`、`README_updated_*` 三类产物。
- `SKILL.md` 列出项目理解文档包路径。
- `phase2_diagrams.md` 要求生成固定理解文档包，同时保留 `phase2_flow_matrix.md` 和 `README_updated_*`。
- 不重新引入“自动证明项目好”“不可伪造”等过度承诺。
